简单了解Tomcat

发表于 2019-02-16
字数统计: 639 | 阅读时长 ≈ 2

在学习的过程中，高频率的见到Tomcat这个词，所以有必要对Tomcat作一定程度上的理解。

首先我们了解一下什么是Tomcat。

Tomcat是一个Web应用服务器，同时也是一个Servlet/JSP的容器。

Tomcat作为一个Servlet容器的作用，主要是把请求对象交给合适的Servlet。然后把返回的响应对象再返回给客户。


图中的Server即代表着Tomcat。

为什么我们要使用Tomcat这个容器呢？容器到底能帮助到我什么？

下面归纳一下容器的主要功能

  1. 通信支持
    容器能为编程人员在servlet和客户之间搭上一栋桥梁，无需自己建立ServerSocket、监听端口、创建流。容器和Web服务器还达成了既定的协议，不必担心Web代码中的API。
  2. 生命周期管理
    容器负责创建servlet，也负责销毁servlet。**加载类、实例化、初始化servlet、调用servlet方法，垃圾回收等，都是由容器负责
  3. 多线程支持
    容器负责为每个请求建立线程，并管理线程的运作。
  4. 声明方式实现安全
    可以使用XML部署描述文件来配置安全性，易修改。
  5. JSP支持 容器负责把JSP翻译成JAVA

容器如何接受一个请求

  1. 确认是否找的就是Servlet，如果只是单纯的拿取文件就不需要用到servlet。
  2. 建立HttpServletRequest，HttpServletResponse，两个对象。
  3. 找到Servlet，分配线程，传递Req、Res对象给相对应的Servlet
  4. 收到请求对象的Servlet根据请求类型调用响应方法。
  5. 第四条生成的页面会被填在响应对象里面.
  6. 响应对象返回给客户

上面第三点提到了，容器要找到一个特定的Servlet，再去为他分配线程。
出于安全考虑，容器是通过映射来找到Servlet的，每个Servlet可以有三个名称。

  1. 实际类名(完全限定名)
  2. 部署名(内部名)
  3. 公共URL名称(公共URL名)

ps.这样的做法让我们不用暴露目录结构，也更方便后期文件的调整。

容器会在某特定位置按照映射名称查找全部servlet

客户直接看到的是公共的URL名称，部署名是编程人员自个知道的秘密名称。在部署描述文件(DD)中写好之间的映射，如图。
