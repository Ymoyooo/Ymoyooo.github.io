了解Servlet的一生

  1. 用户点击一个URL，指向一个Servlet，容器接收到信息。
  2. 容器创建请求对象、响应对象
  3. 容器寻找特定的Servlet，并加载类
  4. 容器调用Servlet构造函数，并分配线程。
  5. 调用Init()函数，初始化servlet。
  6. 调用service方法，并传递请求、响应对象。
  7. 根据Http方法头选择方法，doGet()/doPost()。
  8. 调用destroy()，回收servlet示例。

了解请求和响应

  1. 了解请求

    1. 每一个请求都在单独的一个线程中运行。
    2. 对于每一个请求都会生成其独有请求、响应对象。
    3. 默认使用Get方法
    4. HttpServletRequest对象由容器实现并提供。

      Get()和Post()方法的区别

      1. Get()请求是幂等的。
      2. Get()请求是可以建立书签的。
      3. Post()请求用于发送数据并作处理，而Get只是获取。

  2. 了解响应

    1. setContentType()和getWriter()比较常用
    2. 使用JSP来完成I/O
    3. 返回的必须要是MIME类型文件。
    4. 分清字节和字符输出。
    5. 通过setHeader(),addHeader()等调整相应首部，如发送错误，增加cookie等。

重定向和请求分派的区别

1. 重定向是通过sendRedirect(aString)来实现，把新请求发给新的URL。
2. 重定向是用户可见的，会改变浏览器地址栏的URL。
3. 重定向是客户来完成工作的。
4. 若响应已提交，则无法使用重定向。
5. 请求分派是通过各司其职来完成工作的，如调用JSP。
6. 请求分派用户不可见。
7. 请求分派是服务端完成工作。

重定向是告诉浏览器访问另外一个URL，而请求分派只是把请求传递给了服务器上的另一个组件。
